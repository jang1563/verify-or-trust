# Code of Conduct

This project follows a simple standard for public research collaboration:
be respectful, precise, and evidence-oriented.

## Expected Behavior

- Discuss benchmark behavior, schemas, results, and artifacts on their technical merits.
- Assume good faith while asking for reproducible evidence.
- Keep disagreement specific: point to files, commands, schemas, substrate rows, or result tables.
- Respect upstream data licenses, safety boundaries, and release policies.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Posting secrets, access tokens, unpublished datasets, identifiable human
  subject data, or large upstream data files.
- Repeatedly derailing issue threads away from the repository's benchmark,
  reproducibility, schema, provenance, or artifact-quality scope.

## Reporting

For ordinary reproducibility, documentation, schema, or provenance issues, use
the GitHub issue templates. For security-sensitive reports or accidental
exposure of materials that should not be public, follow `SECURITY.md`.

Maintainers may edit, hide, or close content that violates these expectations.
